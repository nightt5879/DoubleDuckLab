# Content Replacement Inventory（T-0015）

更新时间：2026-04-23

## 1. 使用原则

- 没有真实素材时，不编造成员、论文、招生邮箱、项目成果或线上 CMS 验收结论。
- 能基于仓库事实改写的内容，改成上线演练、内容维护或内部校验语义。
- 不能确认真实性的内容，保留为待替换项，并在上线前由负责人确认。

## 2. 替换清单

| 范围 | 当前处理 | 上线前必须补齐 |
|---|---|---|
| 站点首页文案 | 已改为 go-live rehearsal 语义 | 若 `DoubleDuckLab` 不是最终品牌，替换 `siteName`、`brand` 与首页 highlights |
| 成员 | 已移除明显假邮箱和不应归属个人的示例链接 | 真实姓名、英文名、身份、状态、方向、头像、邮箱/主页/Scholar 等外链 |
| 项目 | 已移除 `example.com` 外链，并把样例项目改为结构演练语义 | 真实项目标题、负责人、进展、图片、公开链接、论文或代码链接 |
| 论文 | 已把 full-field demo 改为内部渲染 fixture，不再伪装真实论文 | 真实论文标题、作者、会议、摘要、PDF、代码、BibTeX |
| 新闻 | 已新增 `2026-04-23-go-live-rehearsal` 双语新闻，并把旧 fake 新闻改为样例/待确认语义 | 使用 `/admin/` 创建一条真实新闻 PR，并完成预览、审核、合并 |
| 招生与合作 | 已移除假邮箱和假地址，改为待确认 | 真实招生方向、名额、联系人、邮箱、地址、响应周期 |

## 3. 不在本轮修改的边界

- 不新增内容类型。
- 不扩展 Decap CMS 到 `papers`、`members`、`projects` 或 `join`。
- 不更改 URL 结构、路由、frontmatter schema 或环境变量命名。
- 不用无法确认的外部信息替换仓库内容。

## 4. 建议下一次真实内容导入顺序

1. 先补招生与合作联系方式，因为这是公开站点最容易误导访客的位置。
2. 再补成员资料，至少确认负责人和当前在读成员。
3. 然后替换项目与论文，确保每条记录都有负责人确认。
4. 最后用 `/admin/` 新建一条真实新闻，完成 PR 预览和合并上线。
